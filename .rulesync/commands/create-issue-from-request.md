ユーザから要求された「$ARGUMENTS」に対して以下の手順で課題を作成します。

- GitHub CLIでまずはissueを作成する
- 必要に応じてユーザとのやり取りでissueの内容を調整する
- 適切なラベルを付与する

## 手順詳細

### GitHub CLIでまずはissueを作成する

issueの内容は日本語で書いてください。

利用可能なテンプレート：
- backend-ticket.md: バックエンド関連のタスク
- frontend-ticket.md: フロントエンド関連のタスク
- general-ticket.md: 汎用的なタスク（インフラ、環境設定など）
- product-backlog.md: プロダクトバックログ

テンプレートに埋めるために必要な情報がない場合は、都度ユーザーに質問をして確認を得ます。

### ユーザとのやり取りでissueの内容を調整する

ユーザから追加の要求や修正依頼があった場合は、`gh issue edit`コマンドを使用してissueの内容を更新してください。

### 適切なラベルを付与する

`gh label list`コマンドを使用して利用可能なラベル一覧を取得し、issueの内容に応じて適切なラベルを選択してください。

例：
- インフラ・環境関連 → infrastructure
- フロントエンド関連 → frontend
- バックエンド関連 → backend
- 新機能や改善 → enhancement

ラベルの付与は`gh issue edit [issue番号] --add-label "ラベル1,ラベル2"`コマンドで行います。

## その他

- 想定外の作業が発生したり、エラーが見つかった場合はユーザに相談してください。
- ghコマンドが使用できない場合は、`gh`コマンドの有無の確認と`gh auth login`コマンドの実行を促してください。
