ユーザーから要求された「$ARGUMENTS」のDOCSファイルの文字起こしから構造化された議事録を生成して下さい。

```
├── 20251105
│   ├── 20251105_minutes.md // 議事録ファイル
│   └── 20251105_キックオフ.docs // 文字起こしファイル
```

## 手順
- docsファイルから文字起こしを読み取ります。
- その文字起こしファイルから、議事録を生成します。

## 議事録生成ルール
### 1. 基本構造
議事録は以下の構造で作成すること:
MTG/templates/simple_minutes_template.mdを参考にして下さい。

```markdown
# 【日付】会議議事録

## 会議情報
- **日時**: YYYY年MM月DD日（曜日） HH:MM～HH:MM
- **議題**: [会議の主要議題]

## サマリー
[会議全体の要約を3～5行で記載]

## 決定事項まとめ
- [決定事項1]
- [決定事項2]
- [決定事項3]

## TODOアイテム
- [TODO内容]（[担当]）

## 議題と討議内容

### 1. [議題名]
**討議内容**:

### 2. [議題名]
**討議内容**:

## 備考・補足事項
[追加で記録すべき情報があれば記載]
```

### 2. 記載のポイント
#### 意思決定の明確化
- 「〜することに決定」「〜の方向で進める」などの決定事項は必ず記録
- 「〜を検討する」「〜を確認する」などの保留事項も明記
- FM様の承認・確認が必要な事項は特に強調

#### TODOアイテムの具体化
- 誰が（FM様 / サンプル株式会社 / POS様）
- 何を（具体的なTODO内容）
を明確に記載

### 3. 出力形式
- Markdown形式で出力
- 読みやすさを重視し、適切に箇条書きと表を使用
- 重要な決定事項は太字で強調

### 4. 注意事項
- 技術的な専門用語は正確に記載
- 曖昧な表現は避け、具体的に記述
- サンプル株式会社側の人は「さん」づけ。その他は「様」づけ。
